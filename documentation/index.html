Custom commands

add/modify or remove a command or return response of command when given command name or return all commands when empty for more complex commands add $ at the start of the command

access change what level of viewer can access a command can also limit it to a specific user(does not include creator or broadcaster)
  • commands add|edit|access|<empty> <command name> [<empty>]|[(only with add|edit)<x> <y>]|[(only with edit)<old command name> <new command name>]|[remove <x>]
  • use of $ is allowed usage shown in message augments
    • using {user} or {input} more than once will result in duplicate text but you CAN use them more than once if you want
  • commands add <command name> <response> creates a command
    • Example: commands add error not a real error - Adds command called "error".
  • commands edit <command name> <response> edits given command
    • Example: commands edit error just a test nothing wrong - edits "error" responce to "just a test nothing wrong".
  • commands remove <command name> - removes given command
  • commands access <command name> <mod|vip|sub|eve> <true|false> - makes command usable by given acces level
  • commands access <command name> <user> - makes command only usable to given user

Auto replies

add/modify or remove a auto replay or return auto replies response if given auto replay name or return all auto replies when empty
default random chance 30 but chance for the auto replies it can be changed after making it by using the chance augment
  • use of $ is allowed usage shown in message augments
    • {user} DO NOT work in vars and will be removed
  • autoreply add <autoreply name> <response> - creates a auto reply with a random chance of "every one in 30"
    • example: autoreply add error a error has occurred - Adds auto reply called "error"
  • autoreply remove <autoreply name> removes given auto reply
  • edit
    • autoreply edit <autoreply name> <response> - edits a given auto reply massage
      • example: autoreply edit error this might be a error you should check - edits autoreply "error" to "this might be a error you should check"
    • autoreply edit <autoreply name> chance <number> - edits a given auto reply chance
      • example: autoreply edit error chance 60 - edits autoreply "error" random chance to 60

  • default
    • firsttimechattergreeting - first time chatter greeting it sends a message when its a viewers first time chatting
      it CAN NOT BE DELETED but the chance and message can be changed
      aug default
      active false
      message $welcome @{user} be sure to read the {prefix}rules

Variables

add/modify or remove a local variable(LV)(used in code) or return LV response given LV name or return all LVs when empty
  • use of $ is allowed usage shown in message augments
    • {input} and {cv-} DO NOT work in vars and will be removed
  • localvars <var name> - shows all available vars
    • example: localvars <var name>
  • localvars add <var name> <x> - creates a variable
    • example: localvars add error a error has occurred - Adds var called "error"
  • localvars edit <var name> <x> - edits a given var
    • example: localvars edit error this might be a error you should check - edits var "error" to "this might be a error you should check"
  • localvars remove <var name> - removes given var
    • example: localvars remove error

  • defaults

      • value default info
      prefix ! the basis for every command
      comcooldown 1 delay(in minutes) till katbot will recognize a command after another
      (does not apply to mods and broadcaster) help to avoid command spam
      timeoutamount 300 default amount of time(in seconds) a user gets timed out for

Repeating messages

add/modify or remove a repeating message or return repeating message response if given repeating messages name or return all repeating messages when empty, default repeat time is 30 minutes
  • use of $ is allowed usage shown in message augments
    • {user}, {input} DO NOT work in repeating messages and will be removed!
  • repeatmsg add <repeatmsg name> <x> - creates a repeat message that repeats every 30 minutes only if 10 messages have passed in that time otherwise the time restarts
    • example: repeatmsg add error a error has occurred - Adds repeating message called "error"
  • repeatmsg edit
    • example: repeatmsg edit error this might be a error you should check - edits var "error" to "this might be a error you should check"
    • example: repeatmsg edit error time 50 - edits repeatimg message "error" repeat time to 50 minutes
    • example: repeatmsg edit error delay 20 - edits repeatimg message "error" minimum amount of messages required to have passed to 20
  • repeatmsg remove - removes given repeating message
    • example: repeatmsg remove <repeatmsg name>

Message augments

requires $ at the start of a message

adds the ability to augment messages used in "custom commands", "auto replys", "variables" and "repeating messages"
  • example: $the bots name is "{nickname}" - message will appear as "the bots name is "katcbot""
    • augment what it does X is the number of the aug
    in the message starting at 0
    {stmrname} adds broadcaster name to message does not take one
    {user} adds user name(if applicable) in the response does not take one
    {nickname} it puts katcbot. does not take one
    {game} current game name does not take one
    {title} current stream title does not take one
    {input} makes command able to take input from user {afterX}
    {cv-customvarname} allows use of custom variable in message like {cv-testvar} using this in the custom var its calling does nothing and will be removed {cvX}
    {datetime()}
    adds EST(eastern standard time) timestamp by default in 12 hour(hour:minute:second).
    it can take any of the formating codes at the python datetime documentation
    with the adition of Mdo (no % in fromt of it) its the same as %d but the respective st, nd, rd or th ordinal version
    example: {datetime(%A %B Mdo %Y %I:%M:%S:%p)}
    		{datetimeX}
    {upper-{X}} x will be all caps, x can be any message augment does not take one
    {lower-{X}} x will be all lower case, x can be any message augment does not take one
    {silent-{X}} x will be silenced when inside of this augment does not take one

symbol

  • msgfilter symbol <active|messageactive|message|reason|punishment|fail> - symbol spam prevention and punishment
    • value default
    active false
    amount 30
    messageactive true
    message $@{user} please watch the symbol spam soft warning
    reason symbol spam
    punishment soft
    fail soft

caps

  • msgfilter caps <active|messageactive|message|reason|punishment|fail> - CAPS spam prevention and punishment
    • value default
    active false
    amount 60
    messageactive true
    message your message contains {percent}% capital letters! soft warning. watch the caps spam
    reason caps spam
    punishment soft
    fail soft
