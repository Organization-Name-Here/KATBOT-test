guide lines

  • YES PUT THE " yes thay have to be " NOT '
  • every thing outside of these <x> within [ ] is required
  • # is just a comment out it means nothing and every thing on that line after it(including it) should be not be added to the message
    • #except for use in calling a custom command from another channel then the same as above just double ##
  • | = or
  • <x> one input
  • <y> another input
  • <z> a third input
  • <x|y|z> x or y or z
  • [] option
  • [<x> <y|z>] x and y or x and z
  • <viewer> viewer name
  • <empty> nothing for that input
  • None means there is no augments at all, NOT None being the augment
  • () is a note and only there to clarify usage DO NOT add () or anything inside it in message
  • #access hierarchy
    • creator
    • broadcaster
    • moderator
    • vip
    • subscriber
    • everyone

Custom commands

  • commands add|edit|<empty> <command name> [<empty>]|[(only with add|edit)<x> <y>]|[(only with edit)<old command name> <new command name>]|[remove <x>]
    • #add/modify or remove a command or return response of command when given command name or return all commands when empty for more complex commands add $ at the start of the command
    • #access change what level of viewer can access a command can also limit it to a specific user(does not include creator or broadcaster)
    • #using {user} or {input} more than once will result in duplicate text but you CAN use them more than once if you want
    • {stmrname} #adds broadcaster name to message
    • {user} #puts user name of who used the command in the command response
    • {nickname} #it puts katcbot.
    • {game} #puts current game name
    • {title} #puts current stream title
    • {youtube} #puts message through KATCBOTS built in YouTube tittle function, will not work if there is not a YouTube link in message
    • {math-(math)} #usage {math-(math in here)} allows complex math like {math-(((520/4.536)/(19*3.937)**2)*703)} for example (see command "math" for more info)
    • {input} #make command only work with a input from a user (eg CommandName UserInput)
    • {DonLink} #add donation(saved in local vars) link to message
    • {r-aListName} #picks random entry from a list, available lists ("ball8","____","didnt","farewell","greeting","jokes","quotes","iconic","response","touch","maiq", "FM"(Pokemon fighting moves),"Taniks")

Auto replies

  • autoreply add|edit|<empty> <auto replay name> [<empty>]|[(only with add|edit)<chance|empty> <x>]|[remove <x>]
    • #add/modify or remove a arlys(auto replay) or return arlys response if given arlys name or return all arlys when empty, default random chance 30 but chance for the arlys can be changed after making it by using the chance augment, use of $ is allowed usage shown in commands
    • #{youtube} DOES NOT work in arlys messages and will be removed!
    • #{input} special case, it will be the last message sent in chat

    • Variables

  • localvars add|edit|<empty> <local var name> [<empty>]|[(only with add|edit)<x> <y>]|[remove <x>] #add/modify or remove a local variable(LV)(used in code) or return LV response given LV name or return all LVs when empty
    • #examble
      • !localvars edit prefix %
    • "prefix" default:! #command prefix
    • "ComCoolDown" default:1 #command cool down(in minutes) to avoid command spam(does not apply to mods and broadcaster)
    • "TimeOutAmount" default:300(5min) #default amount of time a user gets timed out for(in seconds)
    • "CapsAmount" default:60 #percentage of capital letters allowed in message
    • "SymbolAmount" default:30 #percentage of symbols allowed in message
    • "DonationLink" default:"" #donation link

    • Repeating messages

  • repeatmsg add|edit|<empty> <repeating msg name> [<empty>]|[(only with add|edit)<time|delay|empty> <x>]|[remove <x>]
    • #add/modify or remove a repmsg(repeating message) or return repmsg response if given repmsg name or return all repmsgs when empty, default time is 30 minutes but time for the repmsg can be changed after making it by using the time augment, use of $ is allowed usage shown in commands
    • #{user}, {input}, {math} DO NOT work in repeating messages and will be removed!
    • #{youtube} might not work in repeating messages still needs to be tested
